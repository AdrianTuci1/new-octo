# Artifact Construction & Management

Artifacts are structured outputs generated by AI agents during a conversation. They represent high-level results like code changes, project plans, or visual data.

## 1. Supported Artifact Types

The system categorizes artifacts into four primary types, each with specific metadata:

| Type | Description | Key Metadata |
| :--- | :--- | :--- |
| **PLAN** | A structured set of steps to solve a task. | `document_uid`, `notebook_uid`, `title` |
| **PULL_REQUEST** | A link to a code contribution on GitHub. | `url`, `branch`, `repo`, `number` |
| **SCREENSHOT** | Visual capture of a process or result. | `artifact_uid`, `mime_type`, `description` |
| **FILE** | A downloadable asset or generated code file. | `filepath`, `filename`, `mime_type`, `size_bytes` |

## 2. Construction Workflow

### Phase 1: Generation (Server-Side)
The AI model identifies the need for an artifact and sends a specific `ArtifactEvent` through the response stream. This event contains the initial metadata and unique identifiers.

### Phase 2: Serialization & Storage
Local models receive the event and map it to the `Artifact` enum. 
- **Mapping**: The system uses `serde` with tagged variants (e.g., `#[serde(rename = "PLAN")]`).
- **Persistence**: Artifacts are stored as a JSON array within the conversation's metadata in the local SQLite database.

### Phase 3: Rendering & Interaction
The UI (`ArtifactButtonsRow`) renders specific buttons for each artifact:
- **Plans**: Open in a dedicated document viewer or sync to Warp Drive.
- **PRs**: Open the GitHub URL in the default browser.
- **Screenshots**: Trigger a `LightboxView` that fetches signed URLs for secure viewing.
- **Files**: Trigger a `SaveFilePicker` to download the asset to the user's local filesystem.

## 3. Integration with Warp Drive

Artifacts of type `PLAN` have a special relationship with Warp Drive. Once a plan is finalized, it can be "promoted" to a **Notebook**. 
- The `notebook_uid` field is updated.
- The content becomes editable and shareable with team members.
- This creates a bridge between temporary AI chats and permanent project documentation.

## 4. Technical Reference

- **Location**: `app/src/ai/artifacts/mod.rs`
- **Serialization**: Custom `Deserialize` implementation in `Artifact` enum to handle versioning and URL parsing (e.g., extracting PR numbers from GitHub links).
- **Download Logic**: Handled via `artifact_download.rs`, using `spawn` for non-blocking network requests.
